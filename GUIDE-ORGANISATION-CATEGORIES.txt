## 🔧 **COMMENT ÇA FONCTIONNE ?**

### **1. Téléchargement :**
- Les fichiers sont téléchargés dans le dossier principal `DevInstallers/`
- Chaque fichier garde son nom original

### **2. Décompression automatique :**
- Si la décompression automatique est activée
- Le système détecte le type d'archive (.zip, .7z, .rar, .tar.gz, .tgz)
- Extraction automatique vers le dossier de catégorie approprié

### **3. Organisation par catégorie :**
- **Dossier de catégorie** : Créé automatiquement (ex: `Browsers/`, `Editors/`)
- **Dossier du package** : Créé avec le nom du package (ex: `Google_Chrome_Enterprise_x64/`)
- **Fichiers extraits** : Placés dans le dossier du package

## 🎛️ **CONFIGURATION**

### **Activer la décompression automatique :**
- ✅ **CheckBox "Décompression automatique"** : Activé par défaut
- ✅ **CheckBox "Supprimer archives après décompression"** : Optionnel

### **Désactiver l'organisation :**
- Décochez "Décompression automatique" pour garder les archives originales
- Les fichiers resteront dans le dossier principal sans organisation

### **✅ Organisation claire :**
- **Navigation facile** : Trouvez rapidement vos outils par catégorie
- **Structure logique** : Chaque type d'outil a son espace dédié

### **✅ Installation simplifiée :**

### **✅ Gestion des mises à jour :**
- **Remplacement facile** : Supprimez l'ancien dossier et téléchargez le nouveau

## 🚀 **EXEMPLES D'UTILISATION**

### **Exemple 1 : Téléchargement de Chrome**
1. **Téléchargement** : `googlechromestandaloneenterprise64.msi` dans `DevInstallers/`
2. **Décompression** : Si c'est une archive, extraction vers `DevInstallers/Browsers/Google_Chrome_Enterprise_x64/`
3. **Résultat** : Tous les fichiers de Chrome sont organisés dans leur dossier dédié

### **Exemple 2 : Téléchargement de VS Code**
1. **Téléchargement** : `stable.exe` dans `DevInstallers/`
2. **Décompression** : Extraction vers `DevInstallers/Editors/Visual_Studio_Code_(User_x64_stable)/`
3. **Résultat** : VS Code et tous ses composants sont organisés

## 🔍 **TROUBLESHOOTING**

### **❌ Problème : Dossiers non créés**
- **Vérifiez** que la décompression automatique est activée
- **Vérifiez** que le package a une catégorie valide dans le JSON
- **Vérifiez** les logs pour les erreurs de création de dossiers

### **❌ Problème : Fichiers mal organisés**
- **Vérifiez** que la catégorie dans le JSON correspond à l'organisation souhaitée
- **Vérifiez** que les permissions permettent la création de dossiers
- **Vérifiez** que l'espace disque est suffisant

## 🎯 **BONNES PRATIQUES**

### **1. Noms de catégories :**
- Utilisez des noms **clairs et descriptifs**
- Évitez les **espaces et caractères spéciaux**
- Gardez une **cohérence** dans la nomenclature

### **2. Gestion des mises à jour :**
- **Sauvegardez** les dossiers importants avant remplacement
- **Testez** les nouvelles versions dans des dossiers temporaires
- **Documentez** les changements de configuration

### **3. Nettoyage :**
- **Supprimez** régulièrement les anciennes versions
- **Vérifiez** l'espace disque utilisé
- **Archivez** les versions importantes si nécessaire
